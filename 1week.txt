
# 1week 차 정리

* **버전관리** : 누가, 언제 , 무엇을 수정 하였는지 상태의 정보를 확인하는것
* **git 초기화**: 컴퓨터의 프로젝트를 git이 관리 할 수 있도록 설정하는것
* **add/staging** : 변경된 commit을 반영할 파일을 선택 하는것
* **commit** : 누가, 언제 수정하였는지 에 대한 정보를 알아볼수 있게 적은 메세지
* **commit history**: commit 한 순서대로 의 리스트
* **push**: 로컬 repo 에서 수정한 파일을 원격 repo에 올리는것
* **pull**: 원격 repo 에서 수정한 파일을 원격에서 내 컴퓨터로 받는것
* **clone** : 원격 repo를 내 컴퓨터에 가져와 초기 repo를 세팅 하는것
* **tracking**: 내 컴퓨터(로컬 repo)와 원격(원격 repo)이 서로 연결되는것
